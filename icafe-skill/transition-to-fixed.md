# Bug 流转到已修复 - 指南

本文档提供 Bug 类型卡片流转到"已修复"状态的完整流程。

---

## 推荐方式：使用 plan_transition.py

**强烈推荐**使用 `plan_transition.py` 一次性获取完整流转计划：

```bash
# 一次性获取流转路径、必填字段、执行命令
python scripts/plan_transition.py --space edc-scrum --id <card_id> --to "已修复"
```

输出包含：
- 流转路径（新建 → 已分配 → 已分析 → 已修复）
- 推荐字段值（从样本卡片智能推断）
- 可直接执行的命令
- `_instruction` 字段指导如何处理输出

---

## 状态流转交互规则

`plan_transition.py` 返回的 JSON 包含 `_instruction` 字段，指示 Claude 如何处理输出：

### action: "execute_commands"
直接执行 `commands` 中的命令，无需用户确认。

### action: "adjust_and_confirm_fields"
需要智能调整字段值并交由用户确认：

1. **智能调整字段值**：结合卡片信息和对话上下文，调整推荐字段值
2. **展示确认列表**：按状态分组展示调整后的字段值
3. **收集用户确认**：用户回复"确认"或修改特定字段
4. **执行命令**：根据确认值调整命令参数并执行

**调整原则**：
- 根据卡片标题和描述推断最合适的字段值
- 从对话上下文中提取用户意图
- 确保字段值符合 iCafe 预定义选项

---

## 流转路径

完整流转路径：**新建 → 已分配 → 已分析 → 已修复**

不同状态需要填写不同字段：

| 状态 | 必填字段 |
|------|----------|
| 已分配 | 负责人（可选） |
| 已分析 | Bug分析结论、Bug问题原因、开放域子功能模块 |
| 已修复 | 详见下方字段表格 |

---

## 已修复状态必填字段

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| Bug分析结论 | 单选 | 问题分析结论 | 需专业线修复、长线修复 |
| Bug问题原因 | 文本 | 问题原因描述 | 离线识别 badcase |
| Bug修复方案 | 文本 | 修复方案描述 | 干预离线识别引擎，强制纠错 |
| 修改的影响范围 | 文本 | 修改影响的范围 | 离线Query切换小地图模式 |
| 修复的APK版本 | 文本 | 修复的APK版本号 | 0.5.0 |
| 修复的ROM版本 | 文本 | 修复的ROM版本号 | HALO 0.5.0/C CDX707 R04 |
| 实际修复时间 | 日期 | 格式：YYYY-MM-DD | 2026-02-27 |
| 福特-版本计划 | 单选 | 计划合入的版本 | HALO 0.5.0/C CDX707 R04 |
| 开放域子功能模块 | 单选 | 子功能模块分类 | 其他 |

## 命令模板

### 步骤1：新建 → 已分配

```bash
python scripts/update_card.py --space edc-scrum --id <card_id> \
  --status "已分配" \
  --field "负责人" "用户名" \
  --execute
```

### 步骤2：已分配 → 已分析

```bash
python scripts/update_card.py --space edc-scrum --id <card_id> \
  --status "已分析" \
  --field "开放域子功能模块" "其他" \
  --field "Bug分析结论" "需专业线修复" \
  --field "Bug问题原因" "问题描述" \
  --execute
```

### 步骤3：已分析 → 已修复

```bash
python scripts/update_card.py --space edc-scrum --id <card_id> \
  --status "已修复" \
  --field "开放域子功能模块" "其他" \
  --field "Bug分析结论" "需专业线修复" \
  --field "Bug问题原因" "离线识别 badcase" \
  --field "Bug修复方案" "干预离线识别引擎，强制纠错。后续版本合入。" \
  --field "修改的影响范围" "离线Query切换小地图模式" \
  --field "修复的APK版本" "0.5.0" \
  --field "修复的ROM版本" "HALO 0.5.0/C CDX707 R04" \
  --field "实际修复时间" "2026-02-27" \
  --field "福特-版本计划" "HALO 0.5.0/C CDX707 R04" \
  --execute
```

## 实际案例

### 案例：云端泛化错误修复

**场景**：用户提供的卡片标题为"播放爱奇艺视频，语音切换720P，提示需要开通会员，手动可切换为720P"，用户说明"云端泛化错误"，修复方案为"调整泛化，下发正确语音指令"。

**执行过程**：

```bash
# 1. 获取流转计划
python scripts/plan_transition.py --space edc-scrum --id 356446 --to "已修复"
```

**智能推断的字段值**：

| 字段 | 推断值 | 依据 |
|:-----|:-------|:-----|
| Bug分析结论 | 需专业线修复 | 卡片标题显示功能异常 |
| Bug问题原因 | 云端泛化错误 | 用户输入 |
| Bug修复方案 | 调整泛化，下发正确语音指令 | 用户输入 |
| 修改的影响范围 | 切换视频清晰度 | 从卡片标题推断 |
| 修复的APK版本 | 0.5.0 | 从历史数据推断 |
| 修复的ROM版本 | HALO 0.5.0/C CDX707 R04 | 从历史数据推断 |

**确认并执行流转**：
```
用户回复: "确认"

执行命令:
python scripts/update_card.py --space edc-scrum --id 356446 --status 已分配 --execute
python scripts/update_card.py --space edc-scrum --id 356446 --status 已分析 --field Bug分析结论 需专业线修复 --field Bug问题原因 "云端泛化错误" --execute
python scripts/update_card.py --space edc-scrum --id 356446 --status 已修复 --field Bug分析结论 需专业线修复 --field Bug问题原因 "云端泛化错误" --field Bug修复方案 "调整泛化，下发正确语音指令" --execute
```

## Bug分析结论选项

根据卡片实际情况选择合适的分析结论：

| 结论 | 适用场景 |
|:-----|:--------|
| 需专业线修复 | 需要开发修复的 Bug |
| 长线修复 | 需要长期规划的 Bug |
| 不予解决 | 不需要修复的问题 |
| 不规范/未复现/缺日志 | 无法复现或缺少日志信息 |
| 重复问题 | 已经存在相同问题 |
| 非Bug | 不是 Bug，预期行为或需求问题 |
| 符合设计 | 符合产品设计 |

## 关键要点

1. **流转路径不可跳过**：必须按顺序经过每个状态，不能直接从"新建"跳到"已修复"

2. **字段值必须符合预定义选项**：
   - 单选/多选字段必须从预定义选项中选择
   - 使用 `field_config.py` 查看合法选项

3. **推荐使用 plan_transition.py**：
   - 自动计算流转路径
   - 智能推断推荐字段值
   - 生成可执行命令

4. **利用上下文智能推断**：
   - 根据卡片标题和描述推断最合适的字段值
   - 从对话上下文中提取用户意图
   - 如无法确定，保留原始建议值并标注不确定原因

5. **实际修复时间自动填充**：如用户未指定，使用当前日期（格式：YYYY-MM-DD）

6. **版本号从历史数据推断**：修复的APK版本和ROM版本从历史已修复卡片中推断，提高准确性
