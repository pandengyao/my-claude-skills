# Bug 流转到已关闭（非Bug）- 指南

本文档提供 Bug 类型卡片流转到"已关闭"状态（非Bug）的完整流程。

---

## 推荐方式：使用 plan_transition.py

**强烈推荐**使用 `plan_transition.py` 一次性获取完整流转计划：

```bash
# 获取完整流转路径、必填字段建议、执行命令
python scripts/plan_transition.py --space edc-scrum --id <card_id> --to "已关闭"
```

输出示例：
```json
{
  "transition": {
    "path": ["新建", "已分配", "已分析", "已关闭"],
    "total_steps": 3
  },
  "recommended_fields": { ... },
  "commands": [ ... ]
}
```

---

## 状态流转交互规则

`plan_transition.py` 返回的 JSON 包含 `_instruction` 字段，指示 Claude 如何处理输出：

### action: "execute_commands"
直接执行 `commands` 中的命令，无需用户确认。

### action: "adjust_and_confirm_fields"
需要智能调整字段值并交由用户确认：

1. **智能调整字段值**：结合卡片信息和对话上下文，调整推荐字段值
2. **展示确认列表**：按状态分组展示调整后的字段值
3. **收集用户确认**：用户回复"确认"或修改特定字段
4. **执行命令**：根据确认值调整命令参数并执行

---

## 流转路径

完整流转路径（从新建开始）：

```
新建 → 已分配 → 已分析 → 已关闭
```

如果卡片已在"已分配"或"已分析"状态，则从当前状态开始流转。

## 实际案例

### 案例：关闭 edc-scrum-355812（Bug问题原因为非Bug）

**场景**：卡片从"新建"状态开始，需要流转到"已关闭"，Bug问题原因为"非Bug"

**执行过程**：

```bash
# 1. 获取流转计划
python scripts/plan_transition.py --space edc-scrum --id 355812 --to "已关闭"
```

**智能推断的字段值**：

| 字段 | 推断值 | 依据 |
|:-----|:-------|:-----|
| Bug分析结论 | 非Bug | 用户输入 |
| Bug问题原因 | 非Bug | 用户输入 |
| Bug与用例关联 | 非Bug无需测试用例 | 业务规则 |
| QA标注Bug分析 | 已知问题 | 非Bug场景默认值 |
| 福特-版本计划 | 未选择 | 非福特项目默认值 |
| 福特-特殊测试类型 | 其他 | 默认值 |

**确认并执行流转**：
```
用户回复: "确认"

执行命令:
python scripts/update_card.py --space edc-scrum --id 355812 --status 已分配 --execute
python scripts/update_card.py --space edc-scrum --id 355812 --status 已分析 --field Bug分析结论 非Bug --field Bug问题原因 "非Bug" --execute
python scripts/update_card.py --space edc-scrum --id 355812 --status 已关闭 --field Bug分析结论 非Bug --comment "Bug问题原因：非Bug，关闭此问题" --execute
```

## 智能推断规则

当用户输入"非Bug"或类似表达时，系统会自动推断以下字段值：

| 用户输入 | 智能推断字段 |
|---------|-------------|
| "这不是bug" | Bug分析结论 = 非Bug |
| "非Bug" | Bug分析结论 = 非Bug |
| "非bug关闭" | Bug分析结论 = 非Bug，Bug问题原因 = 非Bug |
| "预期行为" | Bug分析结论 = 符合设计 |

**自动填充的关联字段**：
- `Bug与用例关联` = "非Bug无需测试用例"
- `QA标注Bug分析` = "已知问题"（可配置）

## 关键要点

1. **状态流转必须遵循工作流规则**：不能直接跳转状态，需要按中间状态逐步流转
   - 完整路径：新建 → 已分配 → 已分析 → 已关闭

2. **字段值必须符合预定义选项**：
   - 单选/多选字段必须从预定义选项中选择
   - 使用 `field_config.py validate` 验证字段值

3. **必填字段必须填写**：
   - 使用 `plan_transition.py` 获取完整流转计划和字段建议
   - 使用 `detect_required_fields.py` 检测必填字段

4. **关闭状态需要评论**：流转到"已关闭"状态时必须添加 `--comment` 参数

5. **非Bug类卡片特殊字段**：
   - `Bug分析结论` = "非Bug"
   - `Bug问题原因` = "非Bug" 或具体原因描述
   - `Bug与用例关联` = "非Bug无需测试用例"
   - `QA标注Bug分析` = "已知问题" 或其他合法选项

6. **推荐使用 plan_transition.py**：
   - 自动获取流转路径
   - 智能推断字段建议值
   - 生成可直接执行的命令

7. **利用上下文智能推断**：
   - 根据卡片标题和描述推断最合适的字段值
   - 从对话上下文中提取用户意图
   - 如无法确定，保留原始建议值并标注不确定原因

8. **实际修复时间自动填充**：如用户未指定，使用当前日期（格式：YYYY-MM-DD）
