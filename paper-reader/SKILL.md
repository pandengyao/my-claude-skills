# 论文精读 (Paper Reader)

对学术论文（以AI/ML为主）进行深度阅读，生成全面的结构化阅读笔记，保存为Markdown文件。支持单篇和批量输入。

## 工作流程

```
输入解析 → 多工具提取与交叉验证 → 第一遍速览 → 第二遍精读 → 第三遍批判 → 正确性验证 → 输出生成（含回读验证）
```

**多工具提取**：根据PDF类型智能选择工具组合（详见第二阶段），通过优先级规则融合多源信息，防止信息丢失，生成完整的信息全集后再进入精读流程。

批量输入时，对每篇论文独立执行完整流程，生成一一对应的阅读笔记文件。

---

## 第一阶段：输入解析

### 单篇输入

#### arXiv URL

当用户提供arXiv链接（如 `https://arxiv.org/abs/2401.xxxxx` 或 `https://arxiv.org/pdf/2401.xxxxx`）：

1. 从URL提取论文ID
2. 用WebFetch获取arXiv摘要页，提取标题、作者、摘要等基本信息
3. 构造PDF链接：`https://arxiv.org/pdf/{paper_id}.pdf`
4. 用 `curl -L -o` 下载PDF到 `papers/pdf/` 目录：
```bash
curl -L -o papers/pdf/{paper_id}.pdf https://arxiv.org/pdf/{paper_id}.pdf
```
   （同样适用于GitHub等其他来源的PDF链接）
5. 用Read工具读取PDF内容

#### 本地PDF

当用户提供文件路径：

1. 确认文件存在且为PDF格式
2. 用Read工具直接读取（大型PDF超过10页时分批读取，每次≤20页）

### 批量输入

当用户提供多个链接或文件时（以下形式均支持）：
- 对话中直接贴多个 arXiv URL（每行一个或空格/逗号分隔）
- 对话中列出多个本地 PDF 路径
- 提供一个包含链接列表的文本文件（.txt，每行一个URL或路径）
- 混合提供 arXiv URL 和本地 PDF

处理方式：
1. 先解析出所有论文的链接/路径列表
2. 向用户确认识别到的论文数量和列表
3. **逐篇**执行完整的精读流程（速览→精读→批判→验证→输出）
4. 每篇论文生成独立的阅读笔记文件，文件名与论文一一对应
5. 全部完成后，向用户汇报所有生成的笔记文件路径

### 读取失败处理

```
PDF读取遇到问题。可以尝试：
1. 确认文件路径正确且文件未损坏
2. 对于arXiv论文，可提供abs页面链接，我会同时参考网页信息
3. 如有论文的文本版本（如LaTeX源码），也可以直接提供
```

## 第二阶段：多工具提取与交叉验证（Multi-Tool Extraction）

**核心理念**：根据PDF类型智能选择工具组合，用优先级规则融合多源信息，在提取质量和效率间取平衡。所有Python脚本使用 `~/py312/bin/python` 执行。

### 工具箱

| 工具 | 用途 | 适用场景 | Python包 |
|------|------|----------|---------|
| **Read工具** | Claude原生PDF读取 | 所有PDF（必选） | 内置 |
| **PyMuPDF** | 高精度文字提取 | 所有PDF（必选） | `fitz` |
| **pdf2image + VLM** | 转图片后视觉理解 | 含图表/公式/架构图的页面（必选） | `pdf2image` |
| **pdfplumber** | 表格提取 | Word/排版工具生成的PDF（含线框表格） | `pdfplumber` |
| **camelot** | 专业表格提取 | 同上，与pdfplumber互补 | `camelot` |
| **marker-pdf** | ML全能转换 | 所有PDF（可选，有模型时启用） | `marker` |

### Step 0：PDF 类型判断与工具选择

用 PyMuPDF 快速扫描前几页，判断 PDF 类型：

```python
# ~/py312/bin/python
import fitz
doc = fitz.open("path/to/paper.pdf")
page = doc[0]
text = page.get_text("text")
# 检查特征：LaTeX标记（如 arXiv ID、\begin）、字体嵌入、表格线框
```

**判断规则**：
- **LaTeX学术论文**（arXiv等）→ 核心三件套：Read + PyMuPDF + pdf2image。跳过 pdfplumber/camelot（LaTeX表格无线框，提取率接近0%）
- **Word/排版工具生成的PDF** → 全工具：在核心三件套基础上启用 pdfplumber + camelot（表格有线框，提取效果好）
- **扫描件/图片PDF** → Read（可能失败）+ pdf2image（全页转图，视觉理解为主）+ marker-pdf（OCR能力）

**marker-pdf**：所有类型均可启用，但作为补充而非必选。若 marker 模型未下载或执行失败，跳过不阻塞。

### 提取流程

#### Step 1：Read工具直读（必选）
- 使用Read工具读取PDF全文（大型PDF分批，每次≤20页）
- 记录为 **基准文本 A**

#### Step 2：PyMuPDF 文字提取（必选）
- 运行Python脚本提取文字和结构：
```python
# ~/py312/bin/python
import fitz
doc = fitz.open("path/to/paper.pdf")
for page in doc:
    text = page.get_text("text")
    blocks = page.get_text("dict")  # 带位置和字体信息
```
- 记录为 **文本 B**
- 将PyMuPDF全文保存到 `/tmp/paper_pymupdf_text.txt`，后续作为精读参考源（避免反复调用工具）

#### Step 3：关键页面转图片 + 视觉理解（必选）
- 识别包含重要图表、公式推导、架构图的页面
- **注意**：运行前需确保 poppler 在 PATH 中：
```bash
export PATH="/opt/homebrew/bin:$PATH"  # macOS Homebrew
~/py312/bin/python -c "
from pdf2image import convert_from_path
images = convert_from_path('path/to/paper.pdf', dpi=200, first_page=N, last_page=M)
for i, img in enumerate(images):
    img.save(f'/tmp/paper_page_{i}.png')
"
```
- 用Read工具读取图片，视觉理解图表、公式、架构图
- 记录为 **视觉理解 E**

#### Step 4：pdfplumber 表格提取（仅 Word/排版PDF）
- 仅在 Step 0 判断为非LaTeX PDF时执行：
```python
# ~/py312/bin/python
import pdfplumber
pdf = pdfplumber.open("path/to/paper.pdf")
for page in pdf.pages:
    tables = page.extract_tables()
```
- 记录为 **表格集 C**

#### Step 5：camelot 表格精提取（仅 Word/排版PDF）
- 仅在 Step 4 有结果但不完整时执行：
```python
# ~/py312/bin/python
import camelot
tables = camelot.read_pdf("path/to/paper.pdf", pages="all", flavor="lattice")
if len(tables) == 0:
    tables = camelot.read_pdf("path/to/paper.pdf", pages="all", flavor="stream")
```
- 记录为 **表格集 D**

#### Step 6：marker-pdf 全能转换（可选）
- 使用 `marker_single` CLI 转为Markdown：
```bash
~/py312/bin/marker_single path/to/paper.pdf --output_dir /tmp/marker_output
```
- 读取输出的 Markdown 文件，记录为 **Markdown文本 F**
- 特别关注：marker自动还原的LaTeX公式和表格
- 注意：模型缓存在 `~/Library/Caches/datalab/models/`（macOS），首次运行需下载约3GB模型。若模型未下载或执行失败，跳过此步骤

### 信息融合优先级规则

提取完成后，按以下优先级规则融合（不需要逐cell程序化比对，按优先级采信即可）：

1. **图表和公式** → 视觉理解 E 优先级最高（最接近原始渲染）。文字提取 A/B/F 作为补充
2. **正文文本** → PyMuPDF文本 B 为主（结构最完整），Read文本 A 和 marker文本 F 补充遗漏段落
3. **表格数据** → 若有 pdfplumber/camelot 结果（C/D），以提取精度更高的为准；若无，从文字提取中手工解析或以视觉理解 E 为准
4. **遇到分歧** → 以视觉理解 E 为最终裁判（回源原始PDF图像）
5. **信息补充** → 任何工具独有的非冗余信息，只要不与其他源矛盾，均纳入信息全集

### Context 预算控制

多工具提取会产生大量中间文本，需控制规模防止 context 溢出：

- PyMuPDF 全文保存到 `/tmp/paper_pymupdf_text.txt`，按需分段读取，**不要**一次性全量灌入对话
- pdf2image 仅转换关键页面（含图表/公式/架构图），通常不超过 10 页
- marker 输出保存到 `/tmp/marker_output/`，仅在需要核对公式/表格时按需读取
- 进入精读阶段前，清理中间变量，仅保留融合后的 **信息全集** 摘要（关键数据点、公式、图表语义），而非全部原始提取文本

### 注意事项
- 如果某个工具执行失败，跳过该工具继续，不阻塞整体流程
- 临时文件统一放在 `/tmp/` 下，不污染项目目录
- 所有代码示例中的 `path/to/paper.pdf` 应替换为实际PDF路径

## 第三阶段：第一遍速览（Survey）

快速浏览，建立整体认知框架。重点关注：

- **标题与摘要**：提炼核心主张
- **引言**：明确问题定义和研究动机
- **章节标题与图表**：把握论文结构和关键结果
- **结论**：确认主要贡献声明

完成后填写：论文基本信息、一句话总结、研究动机、核心贡献列表。

---

## 第四阶段：第二遍精读（Comprehension）

逐节深入阅读，重点理解：

### 方法部分
- 问题形式化定义（数学符号、变量含义）
- 模型/算法架构设计及其动机
- **所有公式的完整推导**（逐步展开，不跳步）
- 与已有方法的核心区别

### 实验部分
- 数据集选择及其合理性
- 基线方法（Baselines）的选取是否公平全面
- 评价指标的选择
- 主实验结果及消融实验（Ablation Study）的关键发现
- 实验设置中的重要超参数

### 关键图表
- 对每个重要图表进行解读
- 指出图表揭示的关键趋势或结论

---

## 第五阶段：第三遍批判（Critique）

以审稿人（Reviewer）视角进行批判性分析：

### 创新点评估
- 与相关工作的详细对比（方法层面，非简单罗列）
- 创新性类型判断：全新范式 / 有效组合 / 增量改进
- 创新点是否足够支撑论文的核心主张

### 优势分析
- 方法设计的优雅之处
- 实验的充分性和说服力
- 理论或实践意义

### 局限性分析
- 假设条件是否过强
- 实验覆盖是否有盲区
- 可扩展性（Scalability）问题
- 论文中未讨论但值得注意的局限

### 实践导向提炼
- **算法伪代码**：将核心算法整理为清晰的伪代码（若论文已提供则优化注释）
- **复现要点**：关键实现细节（特殊初始化、训练技巧、数据预处理等）
- **代码资源**：论文提供的开源代码链接、第三方复现仓库、相关工具库

---

## 第六阶段：正确性验证（Verification）

对论文中的公式、数据和结论进行验证，这是区别于普通阅读笔记的关键环节：

### 公式验证
- 检查推导过程是否有跳步或错误
- 验证维度/量纲是否一致
- 检查边界条件和特殊情况是否合理
- 用具体数值代入验证公式的正确性

### 数据验证
- 核对表格中的数据是否与文中描述一致
- 检查百分比计算、数值大小是否合理
- 关注不同表格/图之间数据是否自洽

### 结论验证
- 实验结果是否真正支撑核心结论
- 是否存在过度解读或以偏概全
- 统计显著性是否充分

每项验证结果标注为：✅ 已验证 / ⚠️ 存疑 / ❌ 发现问题

## 重要注意事项

### 内容质量
- **准确性优先**：不确定的内容标注"（待确认）"，绝不编造论文未提及的内容
- **公式完整性**：LaTeX公式确保可正确渲染，每个公式都要配数值示例
- **术语一致性**：全篇保持专业术语翻译一致，首次出现附英文
- **批判性但公正**：指出局限时要具体、建设性，避免空洞批评
- **内容纯粹性**：最终输出的阅读笔记只包含论文本身的内容分析，**不要**包含工具提取过程、多工具验证报告、提取效果对比等与论文内容无关的元信息。多工具提取与交叉验证是内部工作流程，不应暴露给读者。

### 适配不同论文类型

根据论文类型调整精读重点和**裁剪笔记模板**（不需要的章节可省略或简化）：

- **理论型论文**：重点放在定理证明推导、假设条件分析。强化"2.3 公式清单"和"5. 正确性验证"，可简化"3. 实验分析"和"6. 复现指南"
