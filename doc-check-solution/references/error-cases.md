# 文档检查案例库

> 此案例库用于记录文档检查过程中发现的有效错误案例，作为后续检查的参考依据。

## 标点符号案例

### 案例 1: 句末缺少句号
- **错误示例**：系统将自动完成数据备份
- **问题说明**：陈述句末尾缺少句号，语句不完整
- **正确写法**：系统将自动完成数据备份。
- **适用场景**：所有完整的陈述句末尾

### 案例 1-2: 句末缺少句号（文档中间位置）
- **错误示例**：显著提升了国产飞桨框架的生态兼容性与技术竞争力
- **问题说明**：陈述句末尾缺少句号，虽然位于段落中间或分页位置，但句子结构完整，应添加句号
- **正确写法**：显著提升了国产飞桨框架的生态兼容性与技术竞争力。
- **适用场景**：文档中任意位置的完整陈述句，特别是段落末尾、分页位置、表格单元格末尾等容易遗漏的位置

### 案例 2: 标点冗余
- **错误示例**：用户点击提交按钮。。
- **问题说明**：句末出现两个句号，属于标点冗余
- **正确写法**：用户点击提交按钮。
- **适用场景**：所有句末标点位置

### 案例 3: 中英文标点混用
- **错误示例**：该功能支持Windows,Mac和Linux系统。
- **问题说明**：中文句子中使用了英文逗号
- **正确写法**：该功能支持Windows、Mac和Linux系统。
- **适用场景**：中文文档中的标点应使用中文标点符号

### 案例 3-2: 中英文引号混用
- **错误示例**：结合国产芯片与飞桨框架适配'赛马'评测结果
- **问题说明**：中文文档中应使用中文引号""，而非英文单引号''
- **正确写法**：结合国产芯片与飞桨框架适配"赛马"评测结果
- **适用场景**：中文文档中的引号应使用中文引号

### 案例 3-3: 中英文括号混用
- **错误示例**：本课题深入研究了基于抽象语法树(AST)的深度学习代码解析和修改方法
- **问题说明**：中文文档中应使用中文括号"（）"，而非英文括号"()"
- **正确写法**：本课题深入研究了基于抽象语法树（AST）的深度学习代码解析和修改方法
- **适用场景**：中文文档中的括号应使用中文括号，特别是技术术语缩写说明时

### 案例 4: 引号不匹配
- **错误示例**：点击"确定按钮继续操作
- **问题说明**：缺少右引号，引号不成对
- **正确写法**：点击"确定"按钮继续操作
- **适用场景**：所有使用引号的地方

### 案例 5: 顿号与逗号混用
- **错误示例**：系统包含用户管理，权限设置，日志查询功能。
- **问题说明**：并列词语之间应使用顿号而非逗号
- **正确写法**：系统包含用户管理、权限设置、日志查询功能。
- **适用场景**：并列词语或短语之间

## 错别字案例

### 案例 1: 同音字错误
- **错误示例**：系统将自动备分数据到服务器
- **问题说明**："备份"误写为"备分"，"分"应为"份"
- **正确写法**：系统将自动备份数据到服务器
- **适用场景**：数据备份相关描述

### 案例 2: 词语搭配错误（截止 vs 截至）
- **错误示例**：截止 2025年7月31日，该课题应到位资金7300.00万元
- **问题说明**："截止"是动词，表示到某个期限为止；"截至"是介词，用于"截至...日/时"的表述，此处应用"截至"
- **正确写法**：截至 2025年7月31日，该课题应到位资金7300.00万元
- **适用场景**：表示时间范围时使用"截至"，表示动作停止时使用"截止"

### 案例 2-3: 框架名称分隔符不统一
- **错误示例**：支持Caffe/TensorFlow/PyTorch/ONNX四类框架模型向飞桨迁移
- **问题说明**：使用斜杠"/"分隔框架名称，在中文技术文档中不够规范，建议统一使用顿号"、"
- **正确写法**：支持Caffe、TensorFlow、PyTorch、ONNX四类框架模型向飞桨迁移
- **适用场景**：列举多个框架、工具、库等名称时

### 案例 3: 词语搭配错误
- **错误示例**：该功能提高了用户体验的错误率。
- **问题说明**："提高"与"错误率"搭配不当，应使用"降低"
- **正确写法**：该功能降低了用户体验的错误率。
- **适用场景**：描述指标变化时注意动词与名词的搭配

### 案例 3-2: 动词与宾语搭配不当（铺设/奠定基石）
- **错误示例**：更为我国人工智能产业的自主可控发展铺设了重要基石
- **问题说明**：
  1. "铺设"通常搭配"道路、轨道、管道"等具体事物
  2. "基石"是比喻性表达，应该用"奠定"搭配
  3. "奠定基石"或"奠定基础"才是正确的表达
- **正确写法**：更为我国人工智能产业的自主可控发展奠定了重要基石 / 更为我国人工智能产业的自主可控发展奠定了坚实基础
- **适用场景**：检查动词与比喻性宾语的搭配是否恰当

### 案例 4: 专业术语漏字
- **错误示例**：支持自适应的分布式并训练技术
- **问题说明**：
  1. "分布式并训练"漏了一个"行"字，应该是"分布式并行训练"
  2. "并行训练"是深度学习领域的专业术语，指数据/模型并行等分布式训练策略
  3. 漏字后术语不完整，影响专业性
- **正确写法**：支持自适应的分布式并行训练技术
- **适用场景**：技术文档中的专业术语需完整准确，特别关注常见术语的漏字情况

### 案例 4: 代码/宏名称中多余字符
- **错误示例**：通过CINN_BLOCK_REDUCE-_INTERNAL_IMPL等宏组合实现
- **问题说明**：宏名称中多了一个连字符"-"，破坏了代码/宏名称的正确性，可能是编辑时的输入错误
- **正确写法**：通过CINN_BLOCK_REDUCE_INTERNAL_IMPL等宏组合实现
- **适用场景**：技术文档中涉及代码、函数名、宏名、变量名等技术标识符时
- **检查方法**：对文档中的技术标识符进行核对，确保与实际代码/文档一致

### 案例 3-2: 同音字错误导致词义错误（如果 vs 如何）
- **错误示例**：因此如果从国外框架的算法模型快速迁移至国产框架便成为关键难点之一。
- **问题说明**："如果"表示假设条件，"如何"表示方式方法。此处语境是在陈述一个事实或问题，应该用"如何"
- **正确写法**：因此如何从国外框架的算法模型快速迁移至国产框架便成为关键难点之一。
- **适用场景**：陈述问题或难点时，表示"方式方法"的含义应用"如何"，而非表示假设的"如果"

**语义判断规则**：
- 当句子结构为"因此/所以/于是 + [如果/如何] + 动词短语 + 便成为/成为 + 名词（难点/关键/问题）"时
- 如果名词是"难点、关键、问题、挑战"等表示问题性质的词，应该用"如何"
- 如果名词是"条件、假设、前提"等表示假设性质的词，应该用"如果"
- 通过判断句子的语义逻辑来确定用词

### 案例 5: 公司/品牌名称拼写错误
- **错误示例**：基准选择OpenAI、Goolge、Anthropic、阿里、深度求索等大模型公司官网或技术报告上采用的榜单
- **问题说明**："Goolge"拼写错误，正确拼写应为"Google"。公司或品牌名称拼写错误会严重影响文档的专业性和可信度
- **正确写法**：基准选择OpenAI、Google、Anthropic、阿里、深度求索等大模型公司官网或技术报告上采用的榜单
- **适用场景**：技术文档中引用公司、品牌、产品名称时，需仔细核对拼写
- **检查方法**：对文档中的公司名称、品牌名称进行专项检查，可建立常见公司名称列表进行对照

## 内容不通顺案例

### 案例 1: 主语缺失
- **错误示例**：通过优化算法，提高了处理速度。
- **问题说明**：缺少主语，不清楚是谁提高了处理速度
- **正确写法**：通过优化算法，系统提高了处理速度。
- **适用场景**：技术方案描述中需明确主体

### 案例 2: 谓语缺失
- **错误示例**：针对大规模稠密参数模型高效训练问题，飞桨多维混合并行训练策略，即将数据并行、张量模型并行等策略组合使用。
- **问题说明**：
  1. "飞桨多维混合并行训练策略"是主语，但后面直接接逗号和"即..."，缺少谓语动词
  2. 句子结构不完整，缺少核心谓语
- **正确写法**：针对大规模稠密参数模型高效训练问题，飞桨采用了多维混合并行训练策略，即将数据并行、张量模型并行等策略组合使用。
- **适用场景**：检查句子是否包含完整的主谓宾结构

### 案例 2: 逻辑不连贯
- **错误示例**：用户登录后，系统会自动发送邮件，数据库采用MySQL。
- **问题说明**：前后两句话没有逻辑关联，跳跃性大
- **正确写法**：用户登录后，系统会自动发送邮件通知。系统数据库采用MySQL存储用户信息。
- **适用场景**：需要确保前后文逻辑连贯

### 案例 3: 表达歧义
- **错误示例**：删除用户和管理员的权限。
- **问题说明**：歧义表达，不清楚是删除"用户和管理员"的权限，还是删除用户的权限和管理员的权限
- **正确写法**：删除用户权限和管理员权限。/ 删除用户和管理员的权限（需根据实际意图确认）
- **适用场景**：涉及多个对象时需明确修饰关系

### 案例 4: 名词短语表述不完整
- **错误示例**：有效解决了长文本场景下的填充冗余与计算复杂度问题
- **问题说明**：
  1. "填充冗余与计算复杂度"本身不是问题，表述不完整
  2. 应该是"填充冗余问题"和"计算复杂度高的问题"
  3. 或者改写为"填充冗余问题与计算复杂度过高的问题"
- **正确写法**：有效解决了长文本场景下的填充冗余问题与计算复杂度过高的问题 / 有效解决了长文本场景下的填充冗余与计算复杂度高的问题
- **适用场景**：检查"A与B问题"结构中，A和B是否都能与"问题"构成完整表述

### 案例 5: 句式冗余（"为...来获得"）
- **错误示例**：为文心大模型的预训练和精调来获得更好的性能
- **问题说明**：
  1. "为...来获得"句式不通顺，"来"字冗余
  2. 正确句式应为"为...获得"或"来获得"二选一
- **正确写法**：为文心大模型的预训练和精调获得更好的性能
- **适用场景**：检查"为...来..."句式中是否存在冗余字词

### 案例 6: 词语搭配不当（场景应用/应用场景）
- **错误示例**：覆盖了模型开发、训练及在线推理等场景应用
- **问题说明**：
  1. "场景应用"搭配不当，"场景"和"应用"位置颠倒
  2. 正确搭配应为"应用场景"，即"应用"修饰"场景"
- **正确写法**：覆盖了模型开发、训练及在线推理等应用场景
- **适用场景**：检查"场景应用"是否应为"应用场景"

### 案例 7: 语义重复（动词重复）
- **错误示例**：分别采购了203PFLOPS和54PFLOPS采购（稠密FP16算力）
- **问题说明**：
  1. "采购了...采购"语义重复，同一个动词出现两次
  2. 第二个"采购"应该是名词"算力"
- **正确写法**：分别采购了203PFLOPS和54PFLOPS算力（稠密FP16算力）
- **适用场景**：检查句子中是否有重复出现的动词或名词

### 案例 7-2: 词语重复（叠词错误）
- **错误示例**：查询文心大模型预训练初期的训练日志，包含包含文本理解loss、图像理解loss...
- **问题说明**：
  1. "包含包含"是同一个词语连续重复出现，属于编辑时的输入错误
  2. 可能是复制粘贴或输入时的疏忽导致
  3. 这类错误影响文档的专业性和阅读流畅度
- **正确写法**：查询文心大模型预训练初期的训练日志，包含文本理解loss、图像理解loss...
- **适用场景**：检查文档中是否有词语连续重复出现的情况
- **检查方法**：使用正则表达式`(\S+)\1`检测连续重复的词语，或人工逐句检查

### 案例 4: 主语混乱/语义不通
- **错误示例**：通过持续迭代，飞桨不断提升对硬件的亲和性以及框架自身的易用性，方案能和硬件协同开发，也可与框架同时发版与迭代。
- **问题说明**：
  1. 主语混乱：前半句主语是"飞桨"，后半句突然变成"方案"
  2. 语义不通："方案能和硬件协同开发"讲不通，方案是文档/设计，不能和硬件"协同开发"
  3. 应该是"硬件接入方案支持与硬件厂商协同开发"
- **正确写法**：通过持续迭代，飞桨不断提升对硬件的亲和性以及框架自身的易用性，该方案支持与硬件厂商协同开发，也可与框架同时发版与迭代。
- **适用场景**：长句中需保持主语一致，语义表达需符合逻辑

### 案例 5: 技术概念表达错误
- **错误示例**：突破了异构算力性能不移植的难题
- **问题说明**：
  1. "性能不移植"语义不通，性能是指标，不是可以"移植"的对象
  2. 结合上下文，应该是说算子/代码迁移后性能下降的问题
  3. 正确表达应该是"移植性能差"或"移植后性能损失"
- **正确写法**：突破了异构算力移植性能差的难题 / 突破了异构算力移植后性能损失的难题
- **适用场景**：技术文档中涉及"迁移/移植"与"性能"搭配时，需检查语义逻辑

### 案例 6: 动词与宾语语义搭配错误（提升/降低成本）
- **错误示例**：有效提升了国产硬件的大模型部署成本
- **问题说明**：
  1. "提升成本"表示增加成本，与上下文意图相反
  2. 技术优化应该是"降低成本"，而非"提升成本"
  3. 这类错误会导致读者完全误解原文含义
- **正确写法**：有效降低了国产硬件的大模型部署成本
- **适用场景**：描述成本、费用、开销等指标变化时，注意动词方向

### 案例 7: 连词重复使用
- **错误示例**：项目承担单位全面负责各项任务的组织实施和与管理
- **问题说明**：
  1. "和"与"与"都是连词，连续使用造成语义重复
  2. 应该只保留其中一个连词
- **正确写法**：项目承担单位全面负责各项任务的组织实施与管理
- **适用场景**：检查连词使用是否重复或冗余

### 案例 8: 语义重复/内容冗余
- **错误示例**：提升了整体训练效率，提升了文心大模型在国产硬件上的训练效率。
- **问题说明**：
  1. 两句话表达的意思基本相同，属于内容冗余
  2. "整体训练效率"和"文心大模型在国产硬件上的训练效率"本质上是同一件事
  3. 应合并为一句话或删除其中一句
- **正确写法**：提升了文心大模型在国产硬件上的训练效率。/ 有效提升了整体训练效率。
- **适用场景**：检查段落中是否有意思相近或重复的表述

### 案例 9: 句首句尾词语重复
- **错误示例**：实现该模块的关键是实现...
- **问题说明**：
  1. 句首"实现"和句尾"实现"重复，造成语义不通
  2. "实现...是实现"句式结构混乱，应调整表达方式
  3. 正确表达应为"实现某事的关键在于..."
- **正确写法**：实现该模块的关键在于...
- **适用场景**：检查句首和句尾是否出现相同词语，特别是动词

### 案例 10: 语义重复（代表指）
- **错误示例**：其中"Turn"代表指单次用户输入与模型响应的交互对
- **问题说明**：
  1. "代表"和"指"意思相近，同时使用造成语义重复
  2. 定义术语或概念时，应选择其中一个表达方式
- **正确写法**：其中"Turn"指单次用户输入与模型响应的交互对 / 其中"Turn"代表单次用户输入与模型响应的交互对
- **适用场景**：定义术语或概念时，避免"代表指"、"是指为"等冗余表达

### 案例 11: 表述混乱（故障中等等）
- **错误示例**：也就是大规模集群训练时故障中等等造成的损耗小
- **问题说明**：
  1. "故障中等等"表述混乱，"中等等"不是正确的表达
  2. 可能是"中等故障等"或"故障等"的笔误
  3. 上下文语境是讨论训练有效率，应指"故障等造成的损耗"
- **正确写法**：也就是大规模集群训练时故障等造成的损耗小
- **适用场景**：检查句子中是否有表述不清或疑似笔误的地方

### 案例 12: 句式冗余（在...报告中）
- **错误示例**：谷歌在技术报告中《Gemini: A Family of Highly Capable Multimodal Models》中指出
- **问题说明**：
  1. "在...报告中"和"中"同时使用，造成句式冗余
  2. 应该是"在...报告中指出"或"...报告中指出"二选一
- **正确写法**：谷歌在技术报告《Gemini: A Family of Highly Capable Multimodal Models》中指出 / 谷歌的技术报告《Gemini: A Family of Highly Capable Multimodal Models》指出
- **适用场景**：检查"在...中"句式中是否存在冗余

## 技术方案风格案例

### 案例 1: 术语不一致
- **错误示例**：用户登录后，账号信息会显示在首页。用户账户包含用户名和密码。
- **问题说明**：同一概念使用了"账号"和"账户"两种表述
- **正确写法**：用户登录后，账户信息会显示在首页。用户账户包含用户名和密码。
- **适用场景**：全文应保持术语统一

### 案例 2: 格式不规范
- **错误示例**：步骤1：打开设置。步骤2：点击保存。
- **问题说明**：步骤编号格式不一致，缺少统一规范
- **正确写法**：
  1. 打开设置。
  2. 点击保存。
- **适用场景**：步骤描述应使用统一的列表格式

### 案例 3: 专业表达不准确
- **错误示例**：系统会把数据存到数据库里。
- **问题说明**：口语化表达，技术文档应使用更专业的表述
- **正确写法**：系统将数据持久化存储至数据库。
- **适用场景**：技术方案文档应使用专业术语

### 案例 4: 非正式缩写（H卡）
- **错误示例**：字节跳动在论文中指出近万张H卡训练有效率为97%
- **问题说明**：
  1. "H卡"是行业内的非正式缩写，指NVIDIA H100/H800 GPU
  2. 正式检验报告或技术文档中应使用完整术语或首次出现时说明
  3. 非正式缩写会影响文档的专业性和可读性
- **正确写法**：字节跳动在论文中指出近万张H100 GPU训练有效率为97%（或首次出现时注明"H卡指NVIDIA H100 GPU"）
- **适用场景**：正式技术文档、检验报告、学术论文中应避免使用非正式缩写

## 专业术语准确性案例（深度学习领域）

### 案例 1: 术语分类错误（ONNX 不是框架）
- **错误示例**：支持Caffe/TensorFlow/PyTorch/ONNX四类框架模型向飞桨迁移
- **问题说明**：ONNX（Open Neural Network Exchange）是开放式的模型交换格式，用于模型序列化和跨框架部署，不是深度学习框架。Caffe、TensorFlow、PyTorch 是深度学习框架，而 ONNX 是模型格式，两者属于不同类别
- **正确写法**：支持Caffe、TensorFlow、PyTorch三类框架模型及ONNX格式模型向飞桨迁移
- **适用场景**：列举技术组件时需准确区分其类型和功能

### 案例 2: 框架与库混淆
- **错误示例**：使用HuggingFace框架进行模型训练
- **问题说明**：HuggingFace Transformers 是一个库（library），不是框架。框架通常指 TensorFlow、PyTorch、飞桨等提供完整训练/推理能力的平台
- **正确写法**：使用HuggingFace Transformers库进行模型训练
- **适用场景**：区分框架、库、工具等不同类型的技术组件

### 案例 3: 编译器与解释器混淆
- **错误示例**：Python编译器将代码转换为字节码
- **问题说明**：Python 是解释型语言，使用的是解释器而非编译器。CPython 解释器将源代码编译为字节码并解释执行
- **正确写法**：Python解释器将代码转换为字节码
- **适用场景**：区分编译型语言和解释型语言的处理方式

### 案例 4: 训练与推理混淆
- **错误示例**：使用训练好的模型进行训练
- **问题说明**：训练好的模型应该用于推理或预测，而非训练。训练是模型学习参数的过程，推理是使用训练好的模型进行预测的过程
- **正确写法**：使用训练好的模型进行推理 / 使用训练好的模型进行预测
- **适用场景**：区分模型训练和推理两个阶段

### 案例 5: 参数与超参数混淆
- **错误示例**：调整学习率参数
- **问题说明**：学习率是超参数，不是模型参数。参数是模型训练过程中学习的权重，超参数是训练前设定的配置
- **正确写法**：调整学习率超参数
- **适用场景**：区分模型参数和超参数的概念

### 案例 6: 技术术语大小写不规范
- **错误示例**：昆仑芯和天数智芯应用了WINT4/WINT8量化技术
- **问题说明**：
  1. "WINT4/WINT8"大小写不规范，通常应写作"WInt4/WInt8"或"INT4/INT8"
  2. "INT"表示整数类型，"W"表示权重（Weight），大小写应规范
- **正确写法**：昆仑芯和天数智芯应用了WInt4/WInt8量化技术 / 昆仑芯和天数智芯应用了INT4/INT8量化技术
- **适用场景**：技术文档中的缩写术语需保持大小写规范统一

## 内容重复案例

### 案例 1: 标题重复
- **错误示例**：文档中同一标题在不同段落重复出现
  - 段落107：虚拟设备层开放标准适配接口降低适配成本
  - 段落115：虚拟设备层开放标准适配接口降低适配成本
- **问题说明**：同一标题在文档中重复出现，可能是编辑时复制粘贴导致的疏漏，影响文档结构清晰度
- **正确写法**：删除其中一个重复标题，或根据内容调整为不同的小节标题
- **适用场景**：长文档编辑、多章节报告、技术方案文档
- **检查方法**：使用文档工具的"查找"功能搜索标题文本，检查是否有重复；或提取所有标题后去重对比

### 案例 2: 段落内容重复
- **错误示例**：文档中连续两个段落内容完全相同
- **问题说明**：同一段落内容重复出现，可能是编辑时的复制粘贴错误
- **正确写法**：删除重复段落，保留一份即可
- **适用场景**：所有文档类型
- **检查方法**：使用文本对比工具或脚本检测重复段落
