## 传输方式一览

WebRelay 支持多种文件传输方式：

| 方式 | 说明 | 限制 |
|------|------|------|
| **Lrzsz** | 基于三方工具，点击终端按钮或使用rz/sz命令 | 单文件最大2G；CentOS部分版本支持不友好；大文件可能卡住 |
| **Trzsz** | Lrzsz升级版，体验更好，支持tmux | 需手动安装 |
| **SFTP文件传输** | 基于sftp，文件流不经过relay | 需要机器的账号密码；免密机器可能无法登录 |
| **BST方案** | 借助企业云盘中转文件 | 安全侧提供 |
| **Web服务** | 启动Python HTTP Server | 仅用于下载；注意安全风险，使用完需关闭 |
| **nc命令** | 使用netcat传输 | 需安装nc |
| **剪切板base64** | 适用于小文件、非文本文件 | 操作繁琐 |

### 现象：上传文件不成功，提示skipped
- **原因**：用户无目录修改权限，或该文件已存在
- **解决**：检查/修改当前用户的文件操作权限

### 现象：大文件无法通过lrzsz传输
- **原因**：单文件上限2G。上传表现为无法选中文件，下载表现为被skipped
- **解决**：
  1. 推荐使用SFTP文件传输功能
  2. 打包压缩后传输

### 现象：sz下载文件卡住
- **解决**：
  1. 使用 `sz -be xxx` 替代 `sz -b xxx`（或反之）
  2. 如果卡死无法输入，等待恢复或新建窗口
  3. 推荐使用官方客户端

### 现象：lrzsz下载文件时没有弹出保存路径
- **排查**：
  1. 浏览器是否设置了默认保存路径
  2. 浏览器输入框右侧是否有阻止提示按钮
  3. 浏览器插件导致
  4. 当前登录的是容器（容器不支持传输），需拷贝到开发机后下载

### 现象：下载文件发现内容找不到
- **原因**：浏览器下载目录设置问题
- **解决**：检查浏览器下载目录配置

### 现象：下载文件无内容
- **排查**：检查浏览器设置是否禁用自动下载项；检查浏览器是否阻止了下载

### 现象：点击下载文件无反应
- **解决**：尝试关闭浏览器插件（部分插件会影响页面流程）

## SFTP文件传输问题

### 使用入口
- 左侧收藏节点的「更多」菜单
- 右上角的「文件传输」按钮

### 现象：SFTP登录失败
- **原因**：门神免密登录的机器，文件传输服务器可能无法获取密码
- **解决**：
  1. 尝试获取临时密码（门神密码管理平台，支持root及部分work账号）
  2. 若目标机器无法获取明文密码，可使用自己的开发机进行文件传输，再用scp转发
- **白名单**（如目标服务器有限制）：10.21.2.128、10.21.244.58、10.45.33.111

## 其他传输方式

### Web服务方式（仅下载）
```bash
# 注意机房到办公网开放8000-9000端口
# Python 3
python3 -m http.server 8000
```
浏览器访问 ip:8000 即可下载。**使用完务必关闭，存在安全风险。**

### nc命令方式
```bash
# 上传：服务器监听
nc -l 8000 > file.txt
# 客户端发送
nc ip 8000 < test.txt

# 下载：服务器监听
nc -l 8001 < file.txt
# 客户端接收
nc ip 8001 > test.txt
```

### 剪切板base64方式（适用于小文件）
```bash
# 本地转base64
base64 < Test.class > result.txt
# Mac复制到剪切板
pbcopy < result.txt
# 服务器上还原
base64 -d < result.txt > Test.class
# 用md5验证
md5 Test.class       # 本地
md5sum Test.class    # 服务器
```
